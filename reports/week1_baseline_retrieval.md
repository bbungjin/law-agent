# 1주차 베이스라인 검색 성능 실험

## 실험 개요

- 날짜: 2026-06-19
- 데이터: 법령 조문 1745개 + 판례 129건 (1184 청크) = 총 2929 청크
- 평가셋: QA 25개 (일반인 상담형 질문)
- 청크 전략: 조 단위 (법령) + 판시사항+판결요지 우선 (판례)
- 임베딩 모델: jhgan/ko-sroberta-multitask

## 분석

### 가설
임베딩 검색이 일반인 구어체 질문 → 법률 전문용어 간 어휘 격차를 메워줄 것이므로
BM25보다 높은 성능을 보일 것이다.

### 결과 해석
- **임베딩 > BM25**: 가설 방향은 맞음. 구어체 질문에서 의미 기반 검색의 이점 확인.
- **전체 성능이 낮음 (28%)**: 예상보다 낮은 수치. 원인 분석 필요.

### 낮은 성능의 원인 (초기 분석)

1. **어휘 격차 미해소**: "전세금 못 받음" ↔ "임대차보증금반환"처럼
   일반인 구어체와 법률 용어 간 격차가 여전히 큼.
   → 2주차 Query Rewriting 실험 대상

2. **청크 전략 문제**: 조 단위 청크는 짧고 압축적이어서 임베딩 품질이 낮을 수 있음.
   항목 단위(조+항 통합) 청크가 더 좋을 수 있음.
   → 2주차 청크 전략 실험 대상

3. **판례 수 부족**: 129건은 매우 적은 수. 관련 판례가 없어서 miss가 발생.
   → 판례 추가 수집 또는 수집 키워드 확장 고려

4. **매칭 기준 문제**: 정답 판단 로직이 느슨해서 실제보다 낮게 나올 수 있음.
   수동 확인 필요.
